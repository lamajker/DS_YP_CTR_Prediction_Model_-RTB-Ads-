Модель машинного обучения для прогнозирования вероятности клика (CTR - Click-Through Rate) на рекламное объявление в RTB-аукционе

---

## CTR Prediction Model (RTB Ads)

### О проекте

В данном проекте построена модель машинного обучения для прогнозирования вероятности клика (CTR - Click-Through Rate) на рекламное объявление в RTB-аукционе.

Задача имеет высокую бизнес-ценность, поскольку позволяет:

* оптимизировать ставки в реальном времени;
* повышать эффективность рекламных кампаний;
* увеличивать доход от показов;
* точнее таргетировать пользователей.

С точки зрения машинного обучения это задача бинарной классификации:

* 1 - пользователь кликнул;
* 0 - пользователь не кликнул.

---

### Цель проекта

Разработать модель, которая:

* точно ранжирует объекты по вероятности клика;
* даёт корректные вероятностные оценки;
* устойчива к дисбалансу классов;
* может быть использована в продакшене.

---

### Данные

Используется датасет с признаками рекламных показов, включающий:

* характеристики пользователя;
* параметры аукциона;
* информацию о площадке;
* категориальные и числовые признаки;
* агрегированные (ML) признаки.

Особенность данных - сильный дисбаланс классов, что требует использования специальных метрик и подходов.

### Методология

В проекте реализован единый pipeline, включающий:

* предобработку данных;
* отбор признаков;
* обучение модели.

Все этапы выполняются внутри кросс-валидации, что:

* исключает утечку данных;
* обеспечивает корректную оценку качества;
* гарантирует воспроизводимость результатов.

Отбор признаков также встроен в pipeline и выполняется отдельно на каждом фолде.

---

### Pipeline модели

Финальный pipeline включает:

**Предобработка данных**

* заполнение пропусков;
* масштабирование числовых признаков;
* кодирование категориальных признаков.

**Отбор признаков**

* отбор признаков по корреляции с целевой переменной;
* удаление низковариативных признаков;
* RFECV (Recursive Feature Elimination с кросс-валидацией).

**Модель**

* Logistic Regression.

**Калибровка вероятностей**

* Isotonic Regression.

Все этапы объединены в единый pipeline и выполняются внутри кросс-валидации.

---

### Метрики

Основная метрика:

* PR-AUC (average precision) - оптимальна при дисбалансе классов.

Дополнительные:

* Brier Score - качество вероятностей;
* ECE / MCE - качество калибровки;
* Precision / Recall / F1-score.

---

### Результаты

#### Базовая модель

| Модель          | PR-AUC | Brier |
| --------------- | ------ | ----- |
| DummyClassifier | 0.172  | 0.142 |

#### Финальная модель

| Модель                          | PR-AUC | Brier  | ECE   |
| ------------------------------- | ------ | ------ | ----- |
| LogisticRegression              | 0.3705 | 0.2113 | 0.273 |
| LogisticRegression (calibrated) | 0.3499 | 0.1284 | 0.011 |

---

### Выводы по метрикам

* модель более чем в 2 раза лучше базовой по PR-AUC;
* калибровка значительно улучшает качество вероятностей;
* наблюдается компромисс:

  * снижение PR-AUC (качество ранжирования);
  * улучшение качества вероятностей (Brier, ECE).

Важно: качество ранжирования и калибровка вероятностей являются разными задачами, и их оптимизация может давать противоположный эффект.

---

### Интерпретация модели

Анализ коэффициентов показал:

* наибольшее влияние оказывают признаки, связанные с параметрами аукциона;
* важны характеристики пользователя;
* значимую роль играют категориальные признаки после кодирования.

Модель учитывает совокупность факторов, а не один доминирующий признак.

---

### Калибровка

Проведён анализ калибровки:

* LogisticRegression изначально показывает приемлемую калибровку;
* LinearSVC требует дополнительной калибровки;
* применение Isotonic Regression:

  * значительно улучшает Brier и ECE;
  * делает вероятности пригодными для практического использования.
